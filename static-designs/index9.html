DOCUMENTATION

Frequently asked questions

Common questions about NaturalTTS implementation and usage

How does this tool work? +

NaturalTTS uses Piper, a neural text-to-speech synthesis engine that converts text into natural-sounding speech. The entire processing pipeline runs in your browser:

  1. Text normalization and preprocessing
  2. Grapheme-to-phoneme conversion
  3. Neural network inference via WebAssembly
  4. Audio waveform generation and encoding
Is there a text limit? +

Yes, the current browser implementation supports up to 5,000 characters at once. This is approximately 750-1000 words depending on length. For longer texts, we recommend processing in smaller chunks. Future releases will support batch processing of larger documents.

What file format is the audio downloaded in? +

All generated audio is provided in MP3 format. Technical specifications:

  • Format: MP3 (MPEG Layer-3)
  • Sample rate: 22.05kHz
  • Bit depth: 16-bit PCM
  • Channels: Mono
  • Bitrate: 128kbps
Is my data private? +

Yes, all processing happens locally in your browser. Your text never leaves your device. The technical implementation details:

  • WebAssembly (WASM) module runs the entire TTS pipeline
  • Speech models are downloaded once and cached locally
  • No API calls or data transmission during processing
  • No analytics or tracking in the core application
Can I use the generated audio for commercial purposes? +

Yes, all output generated by NaturalTTS is available for both personal and commercial use under the following conditions:

  • The NaturalTTS engine is released under the MIT license
  • You own the rights to the output you create
  • The voice models are licensed for commercial use
  • Attribution is appreciated but not required

System architecture

// Component Details

Text Parser: Normalizes text, expands abbreviations, handles numbers
TTS Engine: Piper neural TTS model compiled to WebAssembly
MP3 Encoder: LAME encoder compiled to WebAssembly
Voice Models: Quantized 16-bit models, ~20-40MB per voice

// Performance Metrics

Inference speed: ~20x realtime on modern browsers
Memory usage: 120-180MB during synthesis
Initial load: 3-8 seconds for first synthesis
Subsequent processing: <100ms initialization
